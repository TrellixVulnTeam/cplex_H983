CPXERR_Q_NOT_POS_DEF

5002 %s is not convex

#define CPXERR_Q_NOT_POS_DEF 5002

The Q matrix associated with the quadratic objective or with a quadratic constraint must be positive semi-definite for minimizations or negative semi-definite for maximizations. Check the appropriate quadratic term(s). If only the objective is quadratic, you can use the solution target parameter to solve models either to local optimality or to global optimality. For more detail about this choice, see optimality target in the manual Parameters of CPLEX.

This error also arises if a quadratic constraint is not convex. A convex quadratic constraint is of the form: 
 x'Qx + p'x + c <= 0
when Q is positive semi-definite, or 
 x'Qx + p'x + c >= 0
when Q is negative semi-definite.

Quadratic constraints in which the relation is equality (that is, ==) are not convex. In fact, a quadratic equality constraint can be expressed as a <= and >= constraint with the same Q matrix, but that Q matrix cannot be both positive semi-definite in the former constraint and negative semi-definite in the latter. Consequently, quadratic equality constraints can also raise this error.

For more explanation of convexity, semi-definiteness, and quadratic constraints, see the topic in the CPLEX User's Manual Distinguishing between convex and nonconvex QPs.
