CPXXgetbranchcallbackfunc and CPXgetbranchcallbackfunc

The routine CPXXgetbranchcallbackfunc/CPXgetbranchcallbackfunc accesses the user-written callback routine to be called during MIP optimization after a branch has been selected but before the branch is carried out.

int CPXXgetbranchcallbackfunc( CPXCENVptr env, int(CPXPUBLIC **branchcallback_p)(CPXX_CALLBACK_BRANCH_ARGS), void ** cbhandle_p )

int CPXgetbranchcallbackfunc( CPXCENVptr env, int(CPXPUBLIC **branchcallback_p)(CALLBACK_BRANCH_ARGS), void ** cbhandle_p )

Description

Warning:

This is an advanced routine. Advanced routines typically demand a thorough understanding of the algorithms used by CPLEX. Thus they incur a higher risk of incorrect behavior in your application, behavior that can be difficult to debug. Therefore, the team encourages you to consider carefully whether you can accomplish the same task by means of other Callable Library routines instead.

The routine CPXXgetbranchcallbackfunc/CPXgetbranchcallbackfunc accesses the user-written callback routine to be called during MIP optimization after a branch has been selected but before the branch is carried out. CPLEX uses the callback routine to change its branch selection.

Arguments

env

A pointer to the CPLEX environment, as returned by CPXXopenCPLEX/CPXopenCPLEX.

branchcallback_p

The address of the pointer to the current user-written branch callback. If no callback has been set, the returned pointer evaluates to NULL.

cbhandle_p

The address of a variable to hold the user's private pointer.

Return

The routine returns 0 (zero) if successful or an error code otherwise.

Example

 CPXgetbranchcallbackfunc(env, &current_callback,
                          &current_handle);

See also the topic Advanced MIP control interface (legacy) in the CPLEX User's Manual.

For documentation of the user-written callback, see Callback Description in the documentation of CPXXsetbranchcallbackfunc and CPXsetbranchcallbackfunc.
