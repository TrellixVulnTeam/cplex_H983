CPXXtuneparamprobset and CPXtuneparamprobset

The routine CPXXtuneparamprobset/CPXtuneparamprobset tunes the parameters of the CPLEX environment for improved optimizer performance for a set of problems.

int CPXXtuneparamprobset( CPXENVptr env, int filecnt, char const *const * filename, char const *const * filetype, int intcnt, int const * intnum, int const * intval, int dblcnt, int const * dblnum, double const * dblval, int strcnt, int const * strnum, char const *const * strval, int * tunestat_p )

int CPXtuneparamprobset( CPXENVptr env, int filecnt, char const *const * filename, char const *const * filetype, int intcnt, int const * intnum, int const * intval, int dblcnt, int const * dblnum, double const * dblval, int strcnt, int const * strnum, char ** strval, int * tunestat_p )

Description

The routine CPXXtuneparamprobset/CPXtuneparamprobset tunes the parameters of the CPLEX environment for improved optimizer performance for a set of problems. Tuning is carried out by making a number of trial runs with a variety parameter settings. Parameters and associated values which should not be changed by the tuning process (known as the fixed parameters) can be specified as arguments.

This routine does not apply to network models, nor to quadratically constrained programming problems (QCP).

After CPXXtuneparamprobset/CPXtuneparamprobset has finished, the environment will contain the combined fixed and tuned parameter settings, which the user can query or write to a file.

All callbacks, except the tuning callback, will be ignored. Tuning will monitor the value set by CPXXsetterminate/CPXsetterminate and terminate when this value is set.

A few of the parameter settings in the environment control the tuning process. They are specified in the table below; other parameter settings in the environment are ignored.

Parameter Link to documentation Use
CPXPARAM_TimeLimit optimizer time limit in seconds Limits the total time spent tuning
CPXPARAM_DetTimeLimit deterministic time limit Limits the total deterministic time spent tuning
CPXPARAM_Tune_TimeLimit tuning time limit in seconds Limits the time in seconds of each trial run
CPXPARAM_Tune_DetTimeLimit deterministic tuning time limit Limits the time in deterministic ticks of each trial run
CPXPARAM_Tune_Measure tuning measure Controls the tuning evaluation measure
CPXPARAM_Tune_Display tuning information display Controls the level of the tuning display
CPXPARAM_ScreenOutput messages to screen switch Controls screen output

The value *tunestat_p is 0 (zero) when tuning has completed and nonzero when it has not. The three nonzero statuses are CPX_TUNE_ABORT, which will be set when the terminate value passed to CPXXsetterminate/CPXsetterminate is set, CPX_TUNE_TILIM, which will be set when the time limit specified by CPXPARAM_TimeLimit is reached. and CPX_TUNE_DETTILIM, which will be set when the deterministic time limit specified by CPXPARAM_DetTimeLimit is reached. Tuning will set any parameters which have been chosen even when tuning is not completed.

status = CPXtuneparamprobset (env, filecnt, filenames, filetypes,
		       icnt, inum, ival, dcnt, dnum, dval,
		       0, NULL, NULL, &tunestat);

Arguments

env
A pointer to the CPLEX environment, as returned by CPXXopenCPLEX/CPXopenCPLEX.
filecnt
An integer that specifies the number of problem files.
filename
An array of length filecnt containing problem file names.
filetype
An array of length filecnt containing problem file types, as documented in CPXXreadcopyprob/CPXreadcopyprob. May be NULL; then CPLEX discerns file types from the file extensions of the file names.
intcnt
An integer that specifies the number of integer parameters to be fixed during tuning. This argument specifies the length of the arrays intnum and intval.
intnum
An array containing the parameter numbers (unique identifiers) of the integer parameters which remain fixed. May be NULL if intcnt is 0 (zero).
intval
An array containing the values for the parameters listed in intnum. May be NULL if intcnt is 0 (zero).
dblcnt
An integer that specifies the number of double parameters to be fixed during tuning. This specifies the length of the arrays dblnum and dblval.
dblnum
An array containing the parameter numbers (unique identifiers) of the double parameters which remain fixed. May be NULL if dblcnt is 0 (zero).
dblval
An array containing the values for the parameters listed in dblnum. May be NULL if dblcnt is 0 (zero).
strcnt
An integer that specifies the number of string parameters to be fixed during tuning. This specifies the length of the arrays strnum and strval.
strnum
An array containing the parameter numbers (unique identifiers) of the integer parameters which remain fixed. May be NULL if strcnt is 0 (zero).
strval
An array containing the values for the parameters listed in strnum. May be NULL if strcnt is 0 (zero).
tunestat_p
A pointer to an integer to receive the tuning status.

Return

The routine returns 0 (zero) if successful and nonzero if an error occurs.
