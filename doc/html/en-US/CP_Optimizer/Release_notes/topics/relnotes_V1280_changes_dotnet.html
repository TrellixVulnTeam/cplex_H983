Changes for .NET users

Changes in the API for extension engine classes in .NET

The new release introduces some new changes for users of the .NET interface.

New feature: Bound on the objective

CP Optimizer now maintains a bound on the objective (a lower bound for minimization problems and an upper bound for maximization problems). This bound is displayed in the log when a new bound is found, along with the optimality gap when an improving solution has been found. At the end of the search, the best bound and gap are displayed. The methods CP.GetObjBound() and CP.GetObjGap() give access to the bound and the gap. Multi-criterion versions of these are also available through CP.GetObjBounds() and CP.GetObjGaps().

New termination status

A new information field has been added, obtainable from CP.GetInfo(CP.IntParam.SearchStatus). This information indicates whether the search has not been started, is ongoing, has completed, or has been stopped. If the search has been stopped, then the second new information field, CP.IntParam.SearchStopCause delivers the reason for the premature stop of the search. The information field CP.IntParam.FailStatus is deprecated and is replaced by CP.IntParam.SearchStatus and CP.IntParam.SearchStopCause.

Deprecated items

The CP.IntParam.AutomaticReplay parameter is now redundant and has been deprecated. Search goals are now never replayed, but domains are stored when the search exits and are available to you as usual. Any setting of this parameter can be safely removed from your code.

Removal of deprecated items

The methods CP.Solve(ISearchPhase[]) and CP.StartNewSearch(ISearchPhase[]), deprecated since V12.7.0 have now been removed. Instead. use CP.SetSearchPhases(ISearchPhase[]) then either CP.Solve() or CP.StartNewSearch().

The parameter CP.IntParam.PropagationLog, deprecated since V12.6.0, has been removed. There is no direct replacement. For debugging, we suggest:

  • the use of the CP Optimizer log, with the period set to 1 (parameter CP.IntParam.LogPeriod),
  • the use of the conflict refiner for detecting inconsistencies via member function CP.RefineConflict, and
  • for problems with interval variables, the use of failure conflict analysis via the member function CP.ExplainFailure.

The CP.IntParam.ConstraintAggregation parameter, deprecated in V12.5.0, has been removed. It is now part of CP Optimizer presolve, controlled with the parameter CP.IntParam.Presolve.
