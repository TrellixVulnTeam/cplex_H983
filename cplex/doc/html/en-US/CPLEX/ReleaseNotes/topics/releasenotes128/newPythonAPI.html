New features in the Python API of CPLEX V12.8

These release notes highlight new features of interest to Python users.

DOcplex modeling in Python

IBM Decision Optimization CPLEX Modeling for Python, also known as DOcplex, is a native Python modeling library for optimization. It's compatible with the NumPy and pandas libraries. It is available from the PyPI or conda package managers. For more about DOcplex, see the DOcplex landing page.

The following announcements pertain to the Python API of CPLEX V12.8.

Python versions

CPLEX V12.8 supports these versions of Python on platforms that previously supported CPLEX and Python:
  • Python V2.7
  • Python V3.5
  • Python V3.6

Python termination

CPLEX V12.8 offers a new class Aborter to terminate CPLEX optimization in a Python application. For an example of this feature, see the sample mipex4.py distributed with the product.

Indicator constraints in Python API

CPLEX V12.8 offers a new method Cplex.indicator_constraints.add_batch to add multiple indicator constraints to your model simultaneously.

The existing method to add a single indicator constraint to your model Cplex.indicator_constraints.add now accepts an additional argument type to specify the kind of indicator constraint to add to the model.

To query the types of indicator constraints in your model, CPLEX V12.8 offers a new method Cplex.indicator_constraints.get_types.

Runseeds in Python API

CPLEX V12.8 offers a new feature in the Python API to assist you when you evaluate performance and variability of performance in your CPLEX optimization applications. The new feature consists of a method that allows you to run CPLEX with different values for the random seed that supports various internal decisions in CPLEX optimization. See the new method Cplex.runseeds for more detail.

Generic callbacks in Python API

CPLEX V12.8 offers these new features to support generic callbacks in its Python API:
  • a new class cplex.callbacks.Context and its members
    • a new subclass Context.id
    • a new subclass Context.info
    • a new subclass Context.solution_strategy
    • a new method Cplex.callbacks.Context.get_global_lower_bounds
    • a new method Cplex.callbacks.Context.get_global_upper_bounds
    • a new method Cplex.callbacks.Context.get_local_lower_bounds
    • a new method Cplex.callbacks.Context.get_local_upper_bounds
    • a new method Cplex.callbacks.Context.get_relaxation_point
    • a new method Cplex.callbacks.Context.get_candidate_point
    • a new method Cplex.callbacks.Context.get_candidate_ray
    • a new method Cplex.callbacks.Context.is_candidate_incumbent
    • a new method Cplex.callbacks.Context.is_candidate_ray
  • a new method cplex.Cplex.set_callback
See the reference manual of the CPLEX Python API for more detail.

Examples

CPLEX V12.8 includes new Python examples. The example etsp.py models early-tardy scheduling by means of indicator constraints. These examples illustrate features of the new generic callback:
  • admipex8.py
  • admipex9.py
  • bendersatsp2.py

Parameters for network optimization available in Python API

These parameters that support network optimization were formerly available only in other application programming interfaces (APIs) of CPLEX. They are now available in the Python API:

  • Cplex.parameters.network.display documented in the Parameters reference manual by network logging display switch
  • Cplex.parameters.network.iterations documented in the Parameters reference manual by network simplex iteration limit
  • Cplex.parameters.network.netfind documented in the Parameters reference manual by simplex network extraction level
  • Cplex.parameters.network.pricing documented in the Parameters reference manual by network simplex pricing algorithm
  • Cplex.parameters.network.tolerances.feasibility documented in the Parameters reference manual by network primal feasibility tolerance
  • Cplex.parameters.network.tolerances.optimality documented in the Parameters reference manual by network optimality tolerance
