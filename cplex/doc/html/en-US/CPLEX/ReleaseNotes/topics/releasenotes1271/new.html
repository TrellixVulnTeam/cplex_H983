New features in CPLEX V12.7.1

These release notes highlight new features in IBM ILOG CPLEX V12.7.1. Please review these notes before using IBM ILOG CPLEX V12.7.1.

New parameters of CPLEX

CPLEX V12.7.1 offers a new parameter to manage sifting from simplex optimization. For more detail, see sifting from simplex switch in the reference manual Parameters of CPLEX.

Interactive Optimizer and new command line options

When you invoke CPLEX (whether from the command line or from a file of commands), CPLEX V12.7.1 offers a new option that keeps an open session of the Interactive Optimizer active when this new option is used in conjunction with other existing options. For example, the existing option -f specifies a filename to read, and the following sample tells CPLEX to keep the session open for further activity after reading the specified file: 
 
cplex -i -f myfile
For more detail about this option, see the topic Using Interactive Optimizer commands from your operating system in the online manual of the Interactive Optimizer.

Better documentation of error codes about Benders decomposition

Documentation of the error codes CPXERR_NO_DECOMPOSITION and CPXERR_BAD_DECOMPOSITION (introduced in CPLEX V12.7.0) has been improved to distinguish more clearly between a non-existent decomposition and an invalid decomposition of a model for Benders algorithm.

Improved examples for Benders decomposition

Examples showing how to use Benders algorithm in your own applications have been improved in all application programming interfaces (APIs) to illustrate use of annotations to specify a decomposition:

  • In the Callable Library (C API), see benders.c and xbenders.c.
  • In the C++ API, see ilobenders.cpp.
  • In the Java API, see Benders.java.
  • In the .NET API, see Benders.cs.
  • In the Python API, see benders.py.

Reference documentation for elements supporting annotations for a Benders decomposition of a model has also been improved in these respects:

  • In the Callable Library (C API), see the constants declared as CPX_BENDERS_ANNOTATION and CPX_BENDERS_MASTERVALUE.
  • Likewise, in the C++ API, see the constants declared in cpxconst.h as CPX_BENDERS_ANNOTATION and CPX_BENDERS_MASTERVALUE.
  • In the reference manual of the Java API, see IloCplex.CPX_BENDERS_ANNOTATION as well as IloCplex.CPX_BENDERS_MASTERVALUE.
  • In the reference manual of the .NET API, see Cplex.CPX_BENDERS_ANNOTATION as well as Cplex.CPX_BENDERS_MASTERVALUE.
  • In the reference manual of the Python API, see Cplex.long_annotations.benders_annotation as well as Cplex.long_annotations.benders_mastervalue.

Improved examples of PWL

CPLEX V12.7.1 offers more examples illustrating piecewise linear (PWL) constraints:

  • In the Callable Library (C API) see transport.c and xtransport.c.
  • In the C++ API, the existing example of PWL transport.cpp has been renamed as ilotransport.cpp.
  • In the Java API, see the existing example of PWL Transport.java.
  • In the .NET API, see the existing example of PWL Transport.cs.
  • In the Python API, see the existing example of PWL transport.py.
