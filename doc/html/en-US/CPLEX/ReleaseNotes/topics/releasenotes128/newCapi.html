New features in the Callable Library (C API) of CPLEX V12.8

These release notes highlight new features in the Callable Library (C API) in IBM CPLEX V12.8.0.

CPLEX V12.8.0 offers new routines in the Callable Library (C API).

Managing log files

The new routine CPXXsetlogfilename and CPXsetlogfilename opens a file by name for logging CPLEX activity. This new routine replaces the deprecated routine CPXsetlogfile, which has been removed from CPLEX V12.8.0.

The new routine CPXXgetlogfilename and CPXgetlogfilename accesses the name of a log file.

Modeling assistance: same coefficient in objective

CPLEX V12.8.0 supports an additional constant CPXMI_SAMECOEF_OBJ to alert the user when coefficients in the objective are very nearly the same value. See the reference manual of the Callable Library (C API) for documentation of this constant in Modeling information in the CPLEX Callable Library (C API).

Indicator constraints and data checking

CPLEX V12.8 offers more data checking with respect to indicator constraints in your model. CPXXaddindconstraints and CPXaddindconstraints and CPXXaddindconstr and CPXaddindconstr now validate the sense and return an error message in the case of invalid sense when data checking is on. Likewise, these routines return an error if a NaN (not a number) or infinity appear as a righthand side (RHS) value when data checking is on. For more about data checking, see data consistency checking and modeling assistance in the CPLEX Parameters Reference Manual.

New callback routines in the Callable Library (C API)

These new routines in the Callable Library (C API) offer a fresh approach to callbacks for CPLEX, without the limitations imposed by legacy control callbacks in CPLEX. For more about this idea, see the announcement New callback design in CPLEX V12.8.0 in these release notes as well as the documentation of these routines in the reference manual of the Callable Library (C API).
