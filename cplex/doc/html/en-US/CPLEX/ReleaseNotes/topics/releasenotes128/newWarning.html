New warnings in CPLEX V12.8.0

These release notes highlight new warnings available in IBM CPLEX V12.8.0.

These notes announce new warning messages issued by CPLEX V12.8.0

Modeling assistance warns about lack of names

CPLEX V12.8.0 now issues a warning in the absence of names of variables or names of constraints. This new warning appears as information in the CPLEX log. You will see this warning if you invoke modeling assistance on a model that has no names for constraints, for example, when your model refers to constraints only by index.

For more about modeling assistance, see the topic Modeling assistance in CPLEX in the CPLEX User's Manual.

New warning from FeasOpt

CPLEX V12.8.0 issues a new warning message on the designated warning channel if solving the phase II model of FeasOpt fails or stops for unexpected reasons. For example, such a situation can arise in a model with poor numerics or on a platform with insufficient memory for the model. For more about FeasOpt, see Repairing infeasibilities with FeasOpt in the CPLEX User's Manual.
