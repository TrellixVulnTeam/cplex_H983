New error codes in CPLEX V12.8.0

These release notes highlight new error codes in IBM CPLEX V12.8.0.

CPLEX V12.8.0 introduces new error codes.
  • CPXERR_CALLBACK_INCONSISTENT This error arises when your application mixes the new generic callbacks and existing legacy callbacks, thus creating an inconsistent callback configuration.
  • CPXERR_CAND_NOT_RAY This error arises when a generic callback in the candidate context CPX_CALLBACKCONTEXT_CANDIDATE is invoked, but the candidate is not an unbounded ray.
  • CPXERR_CAND_NOT_POINT This error arises when a generic callback in the candidate context CPX_CALLBACKCONTEXT_CANDIDATE is invoked, but the candidate is not a feasible point.
