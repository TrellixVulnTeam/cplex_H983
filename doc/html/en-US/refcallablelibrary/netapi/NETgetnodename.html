CPXXNETgetnodename and CPXNETgetnodename

Accesses the names of a range of nodes in a network stored in a network problem object.

int CPXXNETgetnodename( CPXCENVptr env, CPXCNETptr net, char ** nnames, char * namestore, CPXSIZE namespc, CPXSIZE * surplus_p, CPXDIM begin, CPXDIM end )

int CPXNETgetnodename( CPXCENVptr env, CPXCNETptr net, char ** nnames, char * namestore, int namespc, int * surplus_p, int begin, int end )

Description

The routine CPXXNETgetnodename/CPXNETgetnodename accesses the names of a range of nodes in a network stored in a network problem object. As arguments to this routine, you must specify the beginning of the range, the end of the range, and the length of the array in which this routine returns the node names.

Note:

If the routine returns CPXERR_NEGATIVE_SURPLUS, then the value in *surplus is negative and specifies the difference between the memory provided and the memory required (in bytes).

Arguments

env
A pointer to the CPLEX environment as returned by CPXXopenCPLEX/CPXopenCPLEX.
net
A pointer to a CPLEX network problem object as returned by CPXXNETcreateprob/CPXNETcreateprob.
nnames
Where to copy pointers to node names stored in the namestore array. The length of this array must be at least (end-begin+1). This routine returns the pointer to the name of node i in nnames[i-begin].
namestore
Array of characters to which this routine copies the specified node names. This array can be NULL if namespc is 0 (zero).
namespc
Length of the namestore array.
surplus_p
Pointer to an integer in which this routine returns the difference between namespc and the number of characters required to store the requested names. A nonnegative value specifies that namespc was sufficient. A negative value specifies that it was insufficient. In that case, this routine returns CPXERR_NEGATIVE_SURPLUS, and the negative value of surplus_p specifies an upper limit on the amount of insufficient space in the array namestore.
begin
Index of the first node for which a name is to be obtained.
end
Index of the last node for which a name is to be obtained.

Return

The routine returns 0 (zero) on success and nonzero if an error occurs. The value CPXERR_NEGATIVE_SURPLUS specifies that there was not enough space in the array namestore to hold the names.

Example

 status = CPXNETgetnodename (env, net, nnames, namestore, namespc,
                             &surplus, 0, nnodes-1);
