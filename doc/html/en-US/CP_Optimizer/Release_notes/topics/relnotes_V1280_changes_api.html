Other changes to the API

New termination status

A new information field has been added, obtainable from IloCP::getInfo(IloCP::SearchStatus). This information can indicate that the search has not been started, is ongoing, has completed, or has been stopped. If the search has been stopped, then second new information field, IloCP::SearchStopCause will deliver the cause of the search prematurely stopping. The information field IloCP::FailStatus is now deprecated and replaced by IloCP::SearchStatus and IloCP::SearchStopCause.

Deprecated items

The IloCP::AutomaticReplay parameter is now redundant and has been deprecated. Search goals are now never replayed, but domains are stored when the search exits and are available to you as usual. Any setting of this parameter can be safely removed from your code.

Removal of deprecated items

The methods IloCP::solve(IloSearchPhaseArray) and IloCP::startNewSearch(IloSearchPhaseArray), deprecated since V12.7.0, have now been removed. Please now use IloCP::setSearchPhases(IloSearchPhaseArray) then either IloCP::solve() or IloCP::startNewSearch().

The parameter IloCP::PropagationLog, deprecated since V12.6.0, has been removed. There is no direct replacement. For debugging, we suggest:

  • the use of the CP Optimizer log, with the period set to 1 (parameter IloCP::LogPeriod)
  • the use of the conflict refiner for detecting inconsistencies via member function IloCP::refineConflict, and
  • for problems with interval variables, the use of failure conflict analysis via the member function IloCP::explainFailure .

The IloCP::ConstraintAggregation parameter, deprecated in V12.5.0, has been removed. It is now part of CP Optimizer presolve, controlled with parameter IloCP::Presolve.
