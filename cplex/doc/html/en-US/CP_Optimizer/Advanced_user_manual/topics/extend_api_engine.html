More details on IloCPEngine

During search, one can manipulate the CP engine state using the handle class IloCPEngine.

One technical point you should be aware of, which should not affect you in practice, is that some of the functionality of IloCPEngine is made available through its base class IlcCPEngine. You may use IloCPEngine everywhere without worry, but keep in mind that you will find documentation for some of the functionality that IloCPEngine provides on the IlcCPEngine class.

IlcCPEngine provides:

  • reversibility related API (see saveValue, getHeap, addReversibleAction).
  • the solve method.
  • In-search:
    • reversible IlcConstraint addition to the solved model.
    • control API (fail/exitSearch).
    • random number generation (getRandomInt, getRandomNum).
    • engine state access using methods such as hasObjective(), andgetInfo(...).
  • model exportation in .cpo format.

IloCPEngine derives from IlcCPEngine and provides a way to:

  • access all the extracted model objects (see getAllIloIntVars(), getAllIloIntervalVars(), etc...).
  • map Ilo objects or names to engine entities such as IlcIntVar (see getIntVar(IloIntVar), getIntVar(const char*), etc..).
  • create engine variable domain iterators from IloIntVar handles.
  • introspect the engine state using concert model objects as keys (see isFixed(IloIntVar), getMin(IloIntVar), getObjValue(), etc..).
  • variable domain printing using (see domain(IloNumVar), domain(IloIntervalVar)).
